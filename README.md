# Ankerのプロジェクター 投影サイズシミュレーター

プロフェッショナル仕様のAnkerプロジェクター投影シミュレーションツール。購買決定を支援する直感的でリッチな体験を提供します。

## 🌟 主要機能

### 1. **デュアルビューシミュレーション**
- **📐 上から見た図**: 投影距離と画面サイズの関係を光の広がりとともに可視化（※概算値）
  - 65型TV（基準）を常に表示し、投影画面サイズを相対比較
  - プロジェクターと投影光の表現
  - スマホではフォントサイズを1.4倍に拡大して視認性向上
- **🖼️ 正面から見た図**: 平均的な壁の高さ（2.4m）との比較で実際のサイズ感を体験（※概算値）
  - ドア（2.0m）のシルエット表示
  - 天井との位置関係を確認
  - 大画面時の圧倒的なサイズ感を実感
  - スマホではフォントサイズを1.3倍に拡大、viewBox最適化（viewBoxWidth: 950px, viewBoxStartX: -110px, targetWallWidth: 93%）で見切れを防止

### 2. **動的な明るさシミュレーション**
- プロジェクターの明るさ（ANSIルーメン）に応じてスクリーンの明るさを変化
- 各機種ごとに最適な使用環境を詳細にアドバイス
- 3500ルーメン: 日中の明るいリビングでもカーテン不要
- 1800ルーメン: 昼間利用可能な4Kモデル
- 650ルーメン: カーテンを閉めれば昼間でも利用可能
- 380ルーメン以下: 夜間や暗くした環境を推奨

### 3. **2つの計算モード**
- **投影画面サイズから距離を計算**: 「100インチがほしい」→「2.7m必要」
- **距離から投影画面サイズを計算**: 「3m離せる」→「110インチ可能」
- リアルタイムスライダーで即座に結果を表示

### 4. **視認性の高い寸法表示**
- 半透明バッジで背景に関わらず見やすい表示
- 「投影距離: 2.70m」「画面サイズ: 100インチ」のように項目名付き
- 行間を最適化し、見出し・数値・補足テキストを明確に分離
- 画面の幅×高さも同時表示

### 5. **スマートなエラーハンドリング**
- 最大・最小投影サイズ超過時に即座にアラート
- ビジュアルとテキストで分かりやすく通知
- 対応範囲を明示して代替案を提示
- エラー時も購入ボタンを表示（製品詳細にすぐアクセス可能）

### 6. **リアルタイムフィードバック**
- 部屋の広さアドバイス（必要な畳数を自動計算）
- 明るさに応じた詳細な使用環境のアドバイス
- 公式サイトとAmazonの購入ボタンを横並びで表示
- 製品仕様の即座表示

### 7. **部屋の広さガイド**
- 投影距離と畳数の対応表を表示
- 〜4.5畳から12畳以上まで6段階で分類
- 画面サイズの目安も併記
- 計算ロジックの透明性（奥行き × 1.2、正方形を仮定）

### 8. **モデル比較機能**
- **画面サイズ別 投影距離比較表**: 10サイズ（35-300インチ）で全7機種を一覧比較
  - 各機種の製品画像をヘッダーに表示
  - 「100インチ欲しい」→全機種の必要距離を一目で確認
- **2機種詳細比較**: 2機種を並べてスペックと投影範囲を詳しく比較
  - スペック一覧表示（解像度、明るさ、重量、型など）

### 9. **完全レスポンシブ対応**
- デスクトップとスマホで最適化されたUI
- スマホでは製品画像付きカスタムドロップダウン
- セクション順序の最適化（STEP 1 → STEP 2 → STEP 3）
  - PC: シミュレーション方法 → 画面サイズ → プロジェクター選択
  - スマホ: シミュレーション方法 → プロジェクター選択 → 画面サイズ
- STEPバッジで操作手順を明確化
- 1stビューに主要情報を集約

### 10. **ブランディング**

- Nebulaブランドカラー（#17BBEF）を基調としたデザイン
- 各プロジェクターに推奨サイズ/距離を表示

### 11. **ソーシャルメディア対応**
- OGP（Open Graph Protocol）対応でSNSシェア時に最適化
  - Facebook、Twitter、LINE等で美しいプレビュー表示
  - カスタムOGP画像（ogp.png）を設定
- Vercel Analyticsでアクセス解析可能（ダッシュボードから手動で有効化）

## 🧮 投影サイズ・距離の計算方法

このシミュレーターは**線形補間（Linear Interpolation）**を使用して、正確な投影シミュレーションを実現しています。

### データポイントベースのアプローチ

従来のプロジェクターで使われる「スローレシオ」（固定比率）ではなく、メーカーが実測した正確なデータポイントを使用：

```javascript
// 例：Nebula Cosmos 4K SE
dataPoints: [
    { screenSize: 60, distance: 1.6 },   // 60インチ → 1.6m
    { screenSize: 100, distance: 2.7 },  // 100インチ → 2.7m
    { screenSize: 120, distance: 3.2 }   // 120インチ → 3.2m
]
```

### 線形補間の計算式

**モード1: 画面サイズから距離を計算**

```javascript
// 例：90インチの投影に必要な距離を計算
// 60インチ(1.6m) と 100インチ(2.7m) の間を補間

const ratio = (90 - 60) / (100 - 60)      // 0.75
const distance = 1.6 + 0.75 * (2.7 - 1.6) // 2.425m
```

**モード2: 距離から投影可能な画面サイズを計算**

```javascript
// 例：2.0mの距離で投影可能なサイズを計算
// 1.6m(60インチ) と 2.7m(100インチ) の間を補間

const ratio = (2.0 - 1.6) / (2.7 - 1.6)   // 0.364
const screenSize = 60 + 0.364 * (100 - 60) // 74.5インチ
```

### 範囲モデル（Nebula X1）の補間

X1のようなズーム機能を持つモデルは、距離に幅があるため両端を補間：

```javascript
// データポイント例
{ screenSize: 100, distanceMin: 2.0, distanceMax: 3.3 }

// 両方を線形補間
interpolatedMin = 線形補間(distanceMin)
interpolatedMax = 線形補間(distanceMax)

// 結果：「2.0m - 3.3m」のような範囲表示
```

### 画面の幅・高さ計算

16:9のアスペクト比から、対角線サイズを幅・高さに変換：

```javascript
const aspectRatio = 16 / 9
const widthInches = Math.sqrt(diagonal² / (1 + (1/aspectRatio)²))
const widthCm = widthInches * 2.54  // インチ→cm変換
const heightCm = widthCm / aspectRatio
```

### なぜ線形補間を使うのか？

- **レンズ特性**: Nebulaプロジェクターは完全な直線関係ではないため、実測値の方が正確
- **精度**: メーカー提供の実測データに基づくため信頼性が高い
- **柔軟性**: データポイント間を補間することで、任意のサイズ/距離に対応可能

この方法により、**実測値に基づいた高精度なシミュレーション**を実現しています。

## 📐 部屋の広さガイド

### 投影距離と必要な部屋の広さの関係

| 畳数 | 投影距離 (m) | 画面サイズ (インチ) |
|------|-------------|------------------|
| 〜4.5畳 | 〜2.25m | 40〜85インチ |
| 4.5〜6畳 | 2.25〜2.6m | 85〜100インチ |
| 6〜8畳 | 2.6〜3.0m | 100〜115インチ |
| 8〜10畳 | 3.0〜3.35m | 115〜125インチ |
| 10〜12畳 | 3.35〜3.68m | 125〜140インチ |
| 12畳〜 | 3.68m〜 | 140インチ〜 |

### 計算ロジック

部屋の広さアドバイスは以下のロジックで算出されます：

```javascript
// 1. 必要な部屋の奥行きを計算
//    投影距離の1.2倍を確保（プロジェクター設置スペース + 視聴スペース）
const roomDepthM = throwDistance * 1.2;

// 2. 部屋の面積を計算（正方形の部屋を仮定）
const roomAreaSqM = Math.pow(roomDepthM, 2);

// 3. 畳数に変換（1畳 ≈ 1.62m²）
const estimatedTatami = roomAreaSqM / 1.62;
```

**前提条件：**
- プロジェクターの後ろに設置スペースが必要
- 視聴者の座る位置も考慮
- 正方形の部屋を前提に計算

**具体例：**
- **投影距離 2.7m** → 必要な奥行き 3.24m → 面積 10.5m² → **約7畳**
- **投影距離 3.0m** → 必要な奥行き 3.6m → 面積 13.0m² → **約8畳**
